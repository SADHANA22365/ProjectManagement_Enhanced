# Contributing to Client Project Tracker

Thank you for your interest in contributing to Client Project Tracker! This guide covers the basic workflow for submitting improvements, bug fixes, and documentation updates.

## How to Contribute

1. Fork the repository to your GitHub account.
2. Create a new branch for your work:
   ```bash
   git checkout -b feature/my-update
   ```
3. Make your changes in a focused branch.
4. Run linting before committing:
   ```bash
   npm run lint
   ```
5. Commit your changes with a descriptive message.
6. Push your branch and open a pull request.

## Branch Naming

Use descriptive branch names, such as:

- `feature/add-user-dashboard`
- `fix/auth-error`
- `docs/update-readme`

## Commit Messages

Write clear and concise commit messages. Prefer the format:

- `feat: add new project overview card`
- `fix: resolve login redirect issue`
- `docs: update README with setup instructions`

## Code Style

- Keep code consistent with the existing TypeScript and React style in the project.
- Use clear naming for components, hooks, and utilities.
- Keep component logic easy to read and maintain.

## Documentation Changes

Documentation improvements are welcome. If you update README content, make sure the instructions are accurate and easy to follow.

## Reporting Issues

If you encounter a bug or want to request a feature:

1. Open an issue in the repository.
2. Provide a clear description of the problem or request.
3. Include steps to reproduce the issue when applicable.
4. Share relevant screenshots or error messages.
